Creation of a new WIC form

Inner Workings

WIC Forms rely on data being passed, on creation, from the parent window which opened it.

  • The JWT of the session, to have DB access
  • The Client (our client) ID to access the forms
  • The Form ID for which form we are loading

If the Form loads with the Following data, we can save the user some time

  • The Consultants ID, Who is filling in the form, (this is typically included)
  • The Employee ID, When coming from the Employees link, otherwise will need async, autocomplete
  • Other? At this stage unknown
  • Modules? the Angular7 project uses some modules for permissions, we'll need to see how this affects forms

Once you are sure The above are being set by the Button, The siteVisitForm.html page will build itself in the browser

The Form Config

In the Forms Folder, save a config file with the name matching the FormID, This way the FormBuilder will load the correct config.json

open a json Builder, i.e. https://jsoneditoronline.org/ to ensure your json is clean

Follow the Instructions on how to build a new json, Ensure that you have the correct settings for INPUT fields vs TEXT fields.
